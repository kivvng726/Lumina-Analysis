# 工作流编辑器全链路手动测试指南

## 测试环境准备

### 1. 启动后端服务器
```bash
# 确保后端服务器运行在端口8123
source .venv/bin/activate
python -m workflow_engine.api.server
```

### 2. 启动前端HTTP服务器
```bash
# 在另一个终端启动HTTP服务器（端口8000）
python -m http.server 8000
```

### 3. 打开浏览器
访问：http://localhost:8000/workflow-editor-pro.html

---

## 全链路测试流程

### ✅ 步骤1：创建新工作流

**操作步骤：**
1. 打开 http://localhost:8000/workflow-editor-pro.html
2. 点击左侧蓝色"新建工作流"按钮
3. 在弹出的模态框中输入：
   - 工作流名称：`全链路测试工作流`
   - 描述：`用于测试完整功能的工作流`
4. 点击"创建"按钮

**预期结果：**
- ✅ 模态框打开
- ✅ 可以输入工作流名称
- ✅ 创建后模态框关闭
- ✅ 左侧工作流列表显示新创建的工作流
- ✅ 画布出现一个Start节点
- ✅ 顶部标题显示"全链路测试工作流"
- ✅ 状态徽章显示"已加载"（绿色）

---

### ✅ 步骤2：编辑工作流 - 添加节点

**操作步骤：**
1. 从左侧节点工具栏点击"代码"（Code）按钮
2. 等待画布上出现Code节点
3. 再次点击"代码"按钮，添加第二个Code节点
4. 从节点工具栏点击"结束"（End）按钮

**预期结果：**
- ✅ 画布上出现多个节点
- ✅ 节点类型显示正确：
  - 绿色：Start节点
  - 橙色：Code节点
  - 红色：End节点
- ✅ 每个节点都有标题和类型标签
- ✅ 执行日志显示"添加节点: Code - 代码节点"

---

### ✅ 步骤3：编辑工作流 - 编辑节点属性

**操作步骤：**
1. 点击画布上的Code节点
2. 观察节点被选中（蓝色边框）
3. （可选）查看节点端口（输入/输出端口显示为白色圆点）

**预期结果：**
- ✅ 点击的节点显示蓝色高亮边框
- ✅ 节点显示输入和输出端口
- ✅ 执行日志显示选中节点的信息

**预期结果：**

- ✅ 执行结果面板显示输出

**如果工作流结构不完整：**
- ⚠️  弹出警告："工作流至少需要开始和结束节点"
- ⚠️  弹出警告："工作流必须包含开始和结束节点"

## 🐛 常见问题排查

### 问题1：无法连接到后端

**症状：**
- 保存工作流时显示"保存到后端失败"
- 执行工作流时报错

**解决方案：**
```bash
# 检查后端服务器是否运行
curl http://localhost:8123/health

# 如果没有运行，启动服务器
source .venv/bin/activate
python -m workflow_engine.api.server
```

### 问题2：无法打开前端页面

**症状：**
- 浏览器显示"无法连接"

**解决方案：**
```bash
# 检查HTTP服务器是否运行
curl http://localhost:8000/

# 如果没有运行，启动服务器
python -m http.server 8000
```

### 问题3：工作流执行失败

**症状：**
- 点击执行后状态显示"执行失败"（红色）

**可能原因：**
1. 工作流缺少Start或End节点
2. 节点代码有错误
3. 连接路径不完整

**解决方案：**
- 确保工作流包含Start和End节点
- 检查Code节点的Python代码是否正确
- 确保节点连接形成完整路径（从Start到End）

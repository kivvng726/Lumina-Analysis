# Workflow Standard Format Specification (v1.0)

本文档定义了工作流（Workflow）的标准 JSON 数据结构。该格式作为后端生成与前端画布交互的通用协议。

## 1. 概览 (WorkflowDefinition)

根对象，描述整个工作流的基本信息、全局变量以及结构定义（节点和边）。

```json
{
  "name": "工作流名称 (string, required)",
  "description": "工作流描述 (string, optional)",
  "variables": { ... },
  "nodes": [ ... ],
  "edges": [ ... ]
}
```

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `name` | string | 是 | 工作流的显示名称。 |
| `description` | string | 否 | 简短描述，用于说明工作流的用途。 |
| `variables` | object | 否 | **全局变量**定义。Key 为变量名，Value 为初始值。可在节点中通过 context 访问。 |
| `nodes` | array | 是 | **节点**列表，定义工作流中的所有执行单元。 |
| `edges` | array | 是 | **连线**列表，定义节点之间的流转关系。 |

---

## 2. 节点 (NodeDefinition)

工作流的基本执行单元。

```json
{
  "id": "unique_node_id",
  "type": "LLM",
  "config": { ... }
}
```

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | string | 是 | **节点唯一标识符**。前端画布生成 UUID 或语义化 ID (如 `node_1`)。 |
| `type` | string | 是 | **节点类型**。枚举值：<br>- `Start`: 开始节点 (必须且唯一)<br>- `End`: 结束节点 (必须)<br>- `LLM`: 大模型调用节点<br>- `Code`: Python 代码执行节点<br>- `Condition`: 条件判断/分支节点<br>- `Loop`: 循环控制节点 |
| `config` | object | 是 | 节点的具体配置参数，详见下方 `NodeConfig`。 |

---

## 3. 节点配置 (NodeConfig)

包含节点的 UI 展示信息、Agent 角色设定以及执行参数。

### 3.1 通用字段

所有类型节点共有的配置字段。

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `title` | string | 是 | **节点标题**，在画布上显示的名称。 |
| `description` | string | 否 | 节点功能的详细说明。 |
| `agent_role` | string | 否 | (CrewAI) 负责该节点的 Agent 角色名称。 |
| `agent_goal` | string | 否 | (CrewAI) Agent 的目标描述。 |
| `agent_backstory` | string | 否 | (CrewAI) Agent 的背景人设。 |
| `params` | object | 否 | **执行参数**。结构随 `type` 变化，详见下方。 |

### 3.2 参数对象 (params)

#### A. LLM 节点 (type="LLM")

大模型生成任务。

```json
"params": {
  "model": "deepseek-chat",
  "prompt": "请基于 {{context_var}} 生成一份报告...",
  "inputs": {
    "context_var": "$node_id.result.field"
  }
}
```

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `model` | string | 模型名称 (如 `gpt-4`, `deepseek-chat`)。 |
| `prompt` | string | **提示词模板**。支持使用 `{{variable}}` 语法引用输入变量。 |
| `inputs` | object | **变量映射**。定义 Prompt 中变量的数据来源。<br>格式：`"变量名": "引用路径"`。<br>引用路径示例：<br>- `$node_id.result`: 引用上游节点输出<br>- `$context.var`: 引用全局变量 |

#### B. Code 节点 (type="Code")

Python 代码执行任务。

```json
"params": {
  "code": "def main():\n    return {'status': 'ok'}",
  "inputs": {
    "data": "$prev_node.result"
  }
}
```

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `code` | string | **Python 代码**。必须包含一个 `main()` 函数作为入口。 |
| `inputs` | object | **变量映射**。定义的变量将注入到代码执行环境的全局作用域中，可在代码中直接使用。 |

#### C. Condition 节点 (type="Condition")

条件分支节点，根据条件表达式的结果决定工作流的执行路径。

```json
"params": {
  "condition": "$node_id.field > 10",
  "condition_type": "simple"
}
```

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `condition` | string | **条件表达式**。<br>`condition_type="simple"` 时：支持简单比较，如 `$node_id.field > 10`、`$node_id.status == "success"`<br>`condition_type="python"` 时：支持 Python 表达式，如 `node_outputs['node_id']['field'] > 10` |
| `condition_type` | string | 条件类型，可选值：<br>- `simple`: 简单条件（默认）<br>- `python`: Python 表达式 |

**连线规则：**
- Condition 节点需要多条输出连线
- 每条连线的 `condition` 字段对应条件表达式可能返回的值
- 例如：条件表达式返回 `"true"` 或 `"false"`，则需要分别定义 `condition="true"` 和 `condition="false"` 的连线

**示例：**
```json
{
  "id": "check_status",
  "type": "Condition",
  "config": {
    "title": "检查状态",
    "params": {
      "condition": "$prev_node.status == 'success'",
      "condition_type": "simple"
    }
  }
},
{
  "source": "check_status",
  "target": "handle_success",
  "condition": "true"
},
{
  "source": "check_status",
  "target": "handle_failure",
  "condition": "false"
}
```

#### D. Loop 节点 (type="Loop")

循环节点，支持固定次数循环、条件循环和 foreach 循环。

```json
"params": {
  "loop_type": "count",
  "max_iterations": 10
}
```

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `loop_type` | string | 循环类型，可选值：<br>- `count`: 固定次数循环（默认）<br>- `condition`: 条件循环<br>- `foreach`: 遍历循环 |
| `max_iterations` | number | 最大迭代次数（用于 `count` 和 `condition` 类型，防止死循环） |
| `condition` | string | 循环条件表达式（仅用于 `condition` 类型） |
| `input` | any | 输入数据（仅用于 `foreach` 类型，可以是列表或可迭代对象） |

**循环类型说明：**

1. **固定次数循环 (count)**:
```json
{
  "loop_type": "count",
  "max_iterations": 5
}
```
执行 5 次迭代。

2. **条件循环 (condition)**:
```json
{
  "loop_type": "condition",
  "condition": "node_outputs['collector']['count'] < 10",
  "max_iterations": 100
}
```
只要条件为真就继续循环，最多执行 100 次。

3. **Foreach 循环 (foreach)**:
```json
{
  "loop_type": "foreach",
  "input": "$data_collector.results"
}
```
遍历输入数据的每个元素。

**循环输出：**
循环节点会在状态中累积每次迭代的结果：
- `state.loop_outputs[node_id]`: 包含所有迭代结果的列表
- `state.loop_counters[node_id]`: 当前迭代次数计数器

#### E. Start / End 节点

通常无特定参数，`params` 为空对象 `{}`。

---

## 4. 连线 (EdgeDefinition)

定义节点间的流转逻辑。

```json
{
  "source": "node_a",
  "target": "node_b",
  "condition": null
}
```

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `source` | string | 是 | **源节点 ID**。 |
| `target` | string | 是 | **目标节点 ID**。 |
| `condition` | string | 否 | **分支条件值**。仅当源节点为 `Condition` 类型时使用。<br>例如：源节点返回 "yes"，则流向标记为 `condition="yes"` 的连线。 |

---

## 5. 示例文件

```json
{
  "name": "简单的总结工作流",
  "description": "先收集信息，再进行总结",
  "variables": {
    "topic": "AI News"
  },
  "nodes": [
    {
      "id": "start",
      "type": "Start",
      "config": {
        "title": "开始",
        "params": {}
      }
    },
    {
      "id": "writer",
      "type": "LLM",
      "config": {
        "title": "撰写总结",
        "agent_role": "Writer",
        "params": {
          "model": "deepseek-chat",
          "prompt": "Write a summary about {{topic}}",
          "inputs": {
            "topic": "$context.topic"
          }
        }
      }
    },
    {
      "id": "end",
      "type": "End",
      "config": {
        "title": "结束",
        "params": {}
      }
    }
  ],
  "edges": [
    { "source": "start", "target": "writer" },
    { "source": "writer", "target": "end" }
  ]
}
